Revisions
Revisions from ADL SCORM 2004 4th Ed. Test Suite Version 1.1 to ADL SCORM 2004 4th Ed. Test Suite Version 1.1.1
  • LMS Conformance Test:

    • The instructions for LMS Test Content Packages CO-06, CO-09, and CO-13b were updated to display the correct precondition rule of "If obj1 activity progress known and not completed, then skip"


  • Content Package Conformance Test:

    • The Content Package Conformance Test and Manifest Utility Test were updated to enforce that parents of a given element are correct in both name and XML namespace.
Revisions from ADL SCORM 2004 4th Ed. Test Suite Version 1.0 to ADL SCORM 2004 4th Ed. Test Suite Version 1.1
  • LMS Conformance Test:

    • LMS Test Content Package CO-10 was updated to expect "incomplete" as the result for the GetValue() call on "cmi.objectives.n.completion_status."

    • LMS Test Content Packages RU-15b, RU-15c, and RU-15d were updated to correctly use cmi.exit.

    • LMS Test Content Package MS-07 was added to the LMS Conformance Test to test the evaluation of measure greater than against a referenced objective mapped to a global objective.


  • Content Package Conformance Test:

    • The Content Package Conformance Test and Manifest Utility Test were updated to enforce that children of the <imsss:sequencing> element exist only under an <imsss:sequencing> element. If a child of <imsss:sequencing> is found in a location other than as a child of the <imsss:sequencing> element, an error message will be generated.

  • Content Package Conformance Test:

    • The Content Package Conformance Test was updated to validate all XML extension elements against their provided controlling documents (XSDSs). This change will better align the validation behavior of the Test Suite with other available XML parsers and increase SCORM Content Package interoperability.

    • The Content Package Conformance Test and the Manifest Utility Test were updated to enforce that <preConditionRule>, <exitConditionRule>, and <postConditionRule> elements contain 1 and only 1 <ruleConditions> element as a child. The Content Package Conformance Test and Manifest Utility Test will throw an error if 0 or more than 1 <ruleConditions> element(s) is found as a child of <preConditionRule>, <exitConditionRule>, or <postConditionRule>.

    • The Content Package Conformance Test and Manifest Utility Test were updated to enforce that an "ID" attribute is not permitted on a <sequencing> element that is a child of an <item> or <organization> element. The Content Package Conformance Test and Manifest Utility Test will throw an error if an "ID" attribute is found on a <sequencing> element that is a child of an <item> or <organization> element.

    • The Content Package Conformance Test and Manifest Utility Test were updated to enforce that the "targetObjectiveID" attribute of the <MapInfo> element does not contain only white space characters. The Content Package Conformance Test and the Manifest Utility test will throw an error when a "targetObjectiveID" attribute is found to contain only white space characters.

    • To increase the understanding of the best practice recommendation and to increase interoperability of content packages, the Content Package Conformance Test and Manifest Utility Test were updated to handle the existence of (sub)manifests in the following ways:
      • If a (sub)manifest is detected in an imsmanifest.xml file and the main manifest includes a reference into the (sub)manifest, testing will be terminated and a warning will be thrown indicating that the best practice recommendation was not followed. No conformance labels will be reported.
      • If a (sub)manifest is detected in a imsmanifest.xml file but the main manifest does not include a reference into the (sub)manifest, the (sub)manifest will be not be considered for validation and the conformance test will continue.

    • The Content Package Conformance Test and the Manifest Utility Test were updated to enforce that the <adlnav:presentation> element must exist as a child of an <item> element that contains an "identifierref" attribute. In addition, the "identifierref" attribute must reference a <resource> element that has an "adlcp:scormType" attribute with a value of "asset" or "sco". The Content Package Conformance Test and the Manifest Utility test will throw an error if the <adlnav:presentation> element exists and does not follow this requirement.

    • The Content Package Conformance Test and the Manifest Utility Test were updated to enforce that if the "xsi:schemaLocation" is present on the <manifest> element, it must contain all element namespaces used in the XML instance with their corresponding controlling document (XSD). The Content Package Conformance Test and the Manifest Utility Test will throw an error if a namespace is found to be used in the XML instance that is not included in the "xsi:schemaLocation" attribute on the <manifest> element. This change will better align the validation behavior of the Test Suite with other available XML parsers and increase SCORM Content Package interoperability.

    • The Content Package Conformance Test and the Manifest Utility Test were updated to verify that any controlling documents (schemas) referenced in the "xsi:schemaLocation" exist at both the root of the Content Package and at the location specified in the "xsi:schemaLocation" attribute if it is different than the Content Package root. The Content Package Conformance Test and the Manifest Utility Test will throw an error if the controlling documents are not located in both their root and "xsi:schemaLocation"-specified locations.

    • The Content Package Conformance Test and the Manifest Utility Test were updated to enforce that the <adlcp:timeLimitAction>, <adlcp:completionThreshold>, and <adlcp:dataFromLMS> elements exist only as the child of an <item> element which references a <resource> element with an "adlcp:scormType" attribute value of "sco". The Content Package Conformance Test and the Manifest Utility Test will throw an error if the <adlcp:timeLimitAction>, <adlcp:completionThreshold>, and <adlcp:dataFromLMS> elements exist as the child of an <item> element which does not reference a <resource> element with an "adlcp:scormType" attribute value of "sco".

    • The Content Package Conformance Test was updated to include directories named "common", "vocab", "extend", and "unique" when checking a Content Package for files which are not referenced by the imsmanifest.xml file (Excess Baggage). Previously, any files contained inside a directory with one of the above names was excluded from the Excess Baggage check and therefore any unreferenced files contained in those directories were not logged. The Content Package Conformance Test will now throw a warning if Excess Baggage is found in a directory named "common", "vocab", "extend", or "unique".

    • The Content Package Conformance test now stops if any part of the imsmanifest.xml file fails validation. Therefore, upon the completion of the imsmanifest.xml file validation, if a validation error has occurred, the test stops, fails, and no SCOs are tested.

    • The Content Package Conformance test now stops if any part of the imsmanifest.xml file fails validation. Therefore, if a given <item> element is invalid in any way, the test stops and fails.

    • The Content Package Conformance Test and the SCO RTE Test were updated to stop testing any SCO or Content Package when the SCO does not call Initialize within the SCO Timeout Period.

    • The Content Package Conformance Test was updated to enforce that if the <ruleAction> element exists under a <sequencing> element which is under an <organizations> element, the "action" attribute of the <ruleAction> element must not have the value "exitParent". The Content Package Conformance Test will throw an error if a <ruleAction> element with an "action" attribute value of "exitParent" element exists under a <sequencing> element which is under an <organizations> element.

    • The Content Package Conformance Test was updated to enforce that if the <ruleAction> element exists under a <sequencing> element which is under an <organizations> element, the "action" attribute of the <ruleAction> element must not have the value "previous". The Content Package Conformance Test will throw an error if a <ruleAction> element with an "action" attribute value of "previous" element exists under a <sequencing> element which is under an <organizations> element.

    • The Content Package Conformance Test was updated to not decode any ASCII encoded characters contained in the SCO launch parameters of the "parameters" attribute on the <item> element. The displayed and logged messages reflect any encoded characters exactly as they are in the "parameters" attribute.

    • The Content Package Conformance Test was updated to launch and test "asset" type resources to ensure they do not attempt to communicate with the SCORM API or Data Model. If an "asset" resource attempts to communicate with the SCORM API or Data Model, the Content Package Conformance Test will throw a warning message, continue to test the "asset" resource as a "sco" resource, and fail the test.

    • The Content Package Conformance Test was updated to enforce that the value of the <schemaversion> element is "2004 4th Edition". The Content Package Conformance Test will throw a warning if the value of the <schemaversion> element is "2004 3rd Edition".

    • The Content Package Conformance Test and the Manifest Utility Test were updated to correctly process encoded and un-encoded whitespace based on the ADL, IMS, and XML specifications in the following attributes:
      • <manifest> : identifier
      • <item> : identifier
      • <organizations> : default
      • <organization> : identifier
      • <resource> : identifier
      • <sequencing> : ID
      • <sequencing> : IDRef
      • <primaryObjective> : objectiveID
      • <objective> : objectiveID
      • <mapInfo> : targetObjectiveID
      • <ruleCondition> : referencedObjective

    • The Content Package Conformance Test and the Manifest Utility Test were updated to enforce that the "referencedObjective" attribute of the <ruleCondition> element references a valid "objectiveID" inside the given sequencing activity. If a <sequencingCollection> sequencing activity is referenced by the given activity or the "referencedObjective" attribute exists inside an <sequencingCollection> sequencing activity, the validity of the "referencedObjective" attribute is tested after the local and <sequencingCollection> sequencing activities were merged.

    • The Content Package Conformance Test was updated to enforce the correct use of the <adlseq:data> element. In addition, the Content Package Conformance Test been updated to read <adlseq:data> information from the "imsmanifest.xml" file when testing SCOs.

    • The Content Package Conformance Test was updated to enforce the correct use of the <adlseq:objectives> element.

    • The Content Package Conformance Test was updated to enforce the correct use of the <adlcp:completionThreshold> element including its three new attributes "completedByMeasure", "minProgressMeasure", and "progressWeight".

    • The Content Package Conformance Test was updated to obtain the Completion Threshold Value from the "minProgressMeasure" attribute of the <adlcp:completionThreshold> element when testing SCOs.

    • The Content Package Conformance Test was updated to enforce the correct use of whitespace in "identifierref" attributes. The "identifierref" attribute will be validated exactly, whitespace included, as it appears in the XML.

    • The Content Package Conformance Test was updated to enforce that no "href" attributes on <resource> elements contain references to external URLs. Validation will be stopped if a reference to an external URL is found.

    • The Content Package Conformance Test was updated to validate each unique metadata instance in a given Content Package only one time.

    • The Content Package Conformance Test was updated to validate the extension of each file referenced in the Content Package to ensure that it exists and that it is a valid extension.


  • SCO RTE Conformance Utility Test:
    • The SCO RTE Conformance Utility Test changed how to select SCOs to run during the test.
      • Lists of SCO file paths can be saved to/loaded from a .txt file.
      • The saved lists can be edited outside of the SCO RTE interface to include query strings or other additional attributes.
      • The text window can now be edited. Changes made inside this window will be used during the test.
      • This change makes it unnecessary to select single SCO or Multiple SCO test during the test.

      For more detailed instructions on this change see the Sharable Content Object Utility Operational Instructions.

    • The SCO RTE Conformance Utility Test was updated to enforce the correct use of the adl.data Data Model elements. In addition, the "Add Data Map" option was added to the SCO RTE Conformance Utility Test interface to allow users to enter Data Map information to be used when testing the given SCO.

    • The SCO RTE Conformance Utility Test was updated to enforce the correct use of cmi.completion_threshold Data Model element. In addition, the SCO RTE Conformance Utility Test interface was updated to allow users to enter the value of the "minProgressMeasure" attribute of the <adlcp:completionThreshold> element to be used when testing the given SCO.
