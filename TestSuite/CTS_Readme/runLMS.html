Running the LMS Conformance Test
To run the LMS Conformance Test, click the corresponding link from the main Test Suite menu. Java Plug-in Security Warning dialog boxes will be displayed. This is because the Test Suite is reading and writing files from/to the local file system via Java Applets. Click Grant this Session or Grant Always to allow the Test Suite applet to run. Note that clicking Deny will cause the Test Suite software to not function properly.

If this is not the first time the LMS Conformance Test is run, prior to beginning the LMS Conformance Test ensure that the LMS Test Content Packages are fully removed from the LMS system and all global objectives associated with these content packages are removed.

Note: Check the Java Console after starting the LMS Conformance Test. If there are any Java exceptions reported, close all browser windows and start the test again.

LMS Overview
The LMS Conformance Test verifies that a test subject LMS is conformant with SCORM according to the SCORM 2004 4th Ed. Testing Requirements. This test will verify that the LMS is able to:
  • Import a known SCORM 2004 4th Ed.-Conformant Content Package

  • Launch an Asset

  • Launch SCORM 2004 4th Ed.-Conformant SCOs

  • Support the Run-Time Environment Application Program Interface (API) defined in SCORM

  • Support all the data model elements defined in the SCORM 2004 4th Ed. Run-Time Environment Data Model.

  • Support Sequencing and Navigation as defined in SCORM 2004 4th Ed. Sequencing and Navigation book.
Operational Instructions
Running a new LMS Conformance Test
  1. Click the New LMS Conformance Test button in the Testing Options frame.

  2. Provide the LMS Name, LMS Version/Release Number and LMS Vendor/Developer information in the Test Identification Information frame. This information is purely informative and only used as part of the log.

  3. Click Continue.

  4. Add a fictitious learner to the LMS. Enter the Learner Name and Learner ID assigned to this fictitious learner by the LMS into the Learner Name 1 and Learner ID 1 fields in the Add Learner Information frame.

  5. Add a different fictitious learner to the LMS. Enter the Learner Name and Learner ID assigned to this fictitious learner by the LMS into the Learner Name 2 and Learner ID 2 fields in the Add Learner Information frame.

  6. Click Continue.

  7. Select the associated LMS Test Content Package names from the selection box containing the available test packages and click Add to add to the list of courses being tested. If all of the LMS Test Content Packages are not selected, the Test Suite cannot determine conformance and will not issue Conformance Labels.

  8. Click Continue.

  9. If LMS Test Content Package API was imported then enroll the fictitious learner that was entered as Learner Name 1 and Learner ID 1 in LMS Test Content Package API.

  10. If any of the LMS Test Content Packages between DMI and T-01b were imported then enroll the fictitious learner that was entered as Learner Name 2 and Learner ID 2 in all of the remaining LMS Test Content Packages.

    The remaining steps will give specific instructions on running each LMS Test Content Package.

  11. As the test progresses through each SCO, information about the actions being performed and their outcomes will be written to the test log. You will be able to view any errors that were encountered, as well as the step-by-step progression of the test. When the final SCO terminates, an indication of conformance or non-conformance will be displayed in the log frame.

  12. If the test completes successfully, the "Testing Instructions" box in the LMS Test Content Package SCO will provide further instructions. These instructions will indicate how the LMS exits the current LMS Test Content Package. These instructions will also appear in the Test Suite "Instructions" frame.

  13. If the test fails, an error message will be displayed in the Test Suite Log. If you want to retry the current LMS Test Content Package, click Retry or stop testing by clicking Abort. For detailed information about why the test failed, click the View Detailed Log button.

  14. Upon completion of each LMS Test Content Package, exit from the package/course in the LMS. Return to the Test Suite and click Continue. You may also save the current testing session by clicking Save before clicking Continue. If you want to stop testing at any time click Abort.

  15. If a SCO encounters a problem communicating with the LMS, or an unexpected fatal error occurs and the Test Suite is not able to continue, the test will stop. Whenever possible, the Test Suite will attempt to continue when it encounters conformance errors.

Loading a saved LMS Conformance Test Session
  1. Click the Load Saved LMS Conformance Test button in the Testing Options frame.

  2. The Load LMS Conformance Test frame will display a list of saved LMS Conformance Test Sessions. From this list select the session to load by clicking the session name. If you have any doubt as to which session to load, use the information in the Date Last Saved column.
     
    • Note: If you do not want to continue a previous LMS Conformance Test, select the New LMS Conformance Test button at the bottom of the page. This will direct you to begin a new LMS Conformance Test.

  3. Click the OK button in the message box that indicates that the Saved Session has loaded.

    • Note: If the log folder was moved from its original location, the session will not be able to load. If this happens, a popup will display to indicate that the Summary Log cannot be found. Select another saved session, or begin a new session.

  4. When the Test Suite Log from the saved LMS Conformance Test appears, click Continue to proceed with the LMS Conformance Test.
User Interface (UI) Tests
The SCORM 2004 4th Ed. Test Suite includes several new tests in the LMS Conformance Test for testing LMS provided UI Controls. These tests can be categorized as:

User Interface Inspection Tests
A number of LMS Test Content Packages were added or modified to include UI Inspection Tests. The content packages ask a series of questions about the LMS provided UI. When one of these packages is encountered, respond to the questions based on the current state of the LMS UI. Use the following terms to answer the questions:

  • Visible - A UI device exists and is either Enabled or Disabled:
    • Enabled - a UI device can be triggered by the learner to invoke a given navigation event. In context of the TOC, "Enabled" is analogous to "Selectable".
    • Disabled - a UI device cannot be triggered by the learner to invoke a given navigation event. A non-existent (Hidden) UI device is also considered "Disabled".
  • Hidden - No UI device exists. The learner shall not be able to invoke the navigation event corresponding to the hidden UI device.

User Interface Behavior Tests
A number of LMS Test Content Packages were added or modified to include a number of manually navigated LMS Test Content Packages. You will be prompted to sequence through the LMS Test Content Package manually using the LMS provided UI. When one of these packages is encountered, the LMS Test Content Package will indicate the expected navigation event by both popping-up an alert with the instructions and by writing the instructions in the test SCO. If at any time, it is not possible to trigger the specified Navigation Event from the LMS provided UI, then it will not be possible to continue with the LMS Conformance Test. At that time, abort the LMS Conformance Test by clicking Abort. The LMS is not SCORM conformant due to navigation limitations in its provided UI.

Additional Functionality

When the LMS Conformance Test starts, two new buttons will appear, New LMS Conformance Test, and Load Saved LMS Conformance Test.

  • New LMS Conformance Test: Allows you to start a brand new LMS test session.

  • Load Saved LMS Conformance Test: Presents you with a list of saved LMS Test sessions. Clicking the session name will load that saved session and allow you to resume a previously saved test session. If no sessions were saved you will be presented with an empty list and an option to start a new LMS test.

    • Note: If the log folder was moved from its original location the session will not be able to load. If this happens, a popup will appear indicating that the Summary Log cannot be found. Select another saved session, or begin a new session.

The User Interface was redesigned and new functionality was added to simplify testing. The following buttons were added: Continue, Retry, Save, Abort, and View Detailed Log.

  • Continue: Used to continue from one LMS Test Content Package to the next. When an LMS Test Content Package is finished "Continue" will be enabled so you can proceed to the next LMS Test Content Package.

  • Retry: Allows an LMS Test Content Package to be run again.

    • Note: The current LMS Test Content Package must be exited before clicking Retry. If Retry is clicked while the LMS Test Content Package is still running, there could be problems when attempting to restart.

  • Save: Allows you to save a test session and restart it at a later time. Saving the same session more than once will overwrite the old session information.

  • Abort: Ends the test. An error will be written to the log files expressing that the test was aborted.

  • View Detailed Log: Allows you to view the detailed log for the current LMS Test Content Package at any time during the execution of that LMS Test Content Package.

  • Log Legend: Opens a window describing the meaning of the colors and symbols used in the Test Suite log.

Operational Notes
Both Summary and Detailed Logs are automatically saved in the folder:
<install_directory>\SCORM_2004_4th_Ed_TS_V1.1.1\TestSuite\Logs\LMS\<timestamp-test subject name>\

Note: Logs are saved in this folder as a result of a portability enhancement. If you need to move logs, move the entire folder and its contents since they are required to properly render the logs in the browser.

If the LMS Test Content Packages or SCOs encounter a problem communicating with the LMS, or an unexpected fatal error occurs and the testing software is not able to continue, the test will stop. Whenever possible the test software will attempt to continue when conformance errors are encountered.
