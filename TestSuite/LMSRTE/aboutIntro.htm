The following is an excerpt from the ADL SCORM® Conformance Matrix that is covered by the Learning Management System Conformance Test:

Learning Management System (LMS)
Conformance Label:
  • LMS SCORM 2004 4th Edition Conformant
The LMS shall adhere to the Testing Requirements defined for the following Conformance Categories:
  • LMS Run-Time Environment Version 1.1 (LMS RTE 1.1)

  • LMS Content Aggregation Model Version 1.1 (LMS CAM 1.1)

  • LMS Sequencing and Navigation Version 1.1 (LMS SN 1.1)
Conformance Category:
  • LMS RTE 1.1
Requirements Summary: The LMS shall
  • Be able to launch an asset.
  • Be able to launch a known SCORM 2004 4th Edition Conformant Sharable Content Objects (SCO).
  • Provide and expose an API Instance as a Document Object Model (DOM) object that correctly implements all of the API methods.
  • Correctly implement support for all SCORM 2004 4th Edition Run-Time Environment Data Model Elements.
  • Correctly implement support for all SCORM 2004 4th Edition Navigation Data Model Elements.
Conformance Category:
  • LMS CAM 1.1
Requirements Summary: The LMS shall
  • Be able to "import" and process a known conformant SCORM 2004 4th Edition Content Aggregation Application Profile Content Package.
  • Correctly initialize SCORM 4th Edition Run-Time Environment Data Model elements based on information supplied in a Content Package Manifest.
Conformance Category:
  • LMS SN 1.1
Requirements Summary: The LMS shall correctly implement
  • All of the sequencing behaviors defined by the pseudo-code included in the SCORM 4th Edition Sequencing and Navigation (SN) Version 1.1.
  • Support for all SCORM 2004 4th Edition Navigation Data Model Elements.
  • Support for Navigation User Interface requirements.
