Running the Checksum Utility Test
To run the Checksum Utility, click the corresponding link from the main Test Suite menu. A Java Plug-in Security Warning dialog box will appear because the Test Suite is reading files from and writing files to the local file system via a Java Applet. Click Grant this Session or Grant Always to allow the Test Suite applet to run. If you click Deny, the Test Suite software will not function properly.

Checksum Utility Overview

The Checksum Utility Test verifies that the identified content package generated the summary log or the checksum value from the summary log during a successful run of the Content Package Test.

Operational Instructions
  1. Click Browse... and select the content package whose log or checksum value authenticity you want to evaluate. Select only the zip (PIF) format of the content package or the imsmanifest.xml file from an extracted (non-PIF) format of the content package.
  2. Select the method of providing the checksum value, either manually or by choosing the summary log.
  3. If you chose the Checksum Value option, enter the checksum value in the text box as it appeared in the Content Package Test Summary Log. If you chose the Log File option, click Browse... to select the XML summary log file created by the Content Package Test for the given content package.
  4. Click Compare.
  5. The result of the Checksum Utility Test will appear in the box below the Compare button. A message stating "Checksum Successful" indicates that the given checksum value corresponds to the given SCORM content package. A message stating "Checksum Failed" indicates that the given checksum value does not correspond the given content package. This means you cannot assume the package adheres to the SCORM Testing Requirements.
Operational Notes
  • The Checksum Utility Test can be used only with Content Package Test logs created with the SCORM 2004 4th Edition Test Suite Version 1.1 and SCORM 2004 4th Edition Test Suite Version 1.1.1
  • The Checksum Utility Test can be run multiple times with different SCORM content packages and/or checksum values without closing the browser window.
  • The checksum value is generated by applying a hashing algorithm against all the files within the SCORM content package. Any change to the SCORM content package, such as adding, removing, or modifying files will invalidate the checksum value generated from a previous Content Package Test run.
  • If the SCORM content package needs to be modified, for example to remove files marked as excess baggage or to update resource locations within the manifest, retest the package using the Content Package Test to obtain a new checksum value.
