Running the Manifest Utility Test
To run the Manifest Utility Test, click the corresponding link from the main Test Suite menu. A Java Plug-in Security Warning dialog box will appear because the Test Suite is reading and writing files from/to the local file system via a Java Applet. Click Grant this Session or Grant Always to allow the Test Suite applet to run. Clicking Deny will cause the Test Suite software to not function properly.

Manifest Utility Overview
The Manifest Utility Test verifies that the IMS Manifest is conformant with the rules specified in the SCORM 2004 4th Ed. Testing Requirements.

The purpose of this test is to verify that the IMS Manifest test subject:
  • Is named imsmanifest.xml.
  • Is located at the root of the content package.
  • Is well-formed.
  • Validates against the IMS Content Packaging XML Schema Definition (XSD) Version 1.1.4 ( imscp_v1p1.xsd found at http://www.imsglobal.org/ ).
  • Validates against the SCORM 2004 4th Ed. Content Packaging Extensions XML XSD Version 2.0 ( adlcp_v1p3.xsd found at http://www.adlnet.gov/ ).
  • Implements all mandatory elements correctly.
  • Uses optional elements, and if used, that they are used correctly.
  • Uses extensions elements, and if used, that they are valid to their respective controlling document.
  • Verifies that the sequencing information is valid according to the IMS Simple Sequencing XML Schema Definition (XSD) ( imsss_v1p0.xsd and supporting files found at http://www.imsglobal.org/ ) if the IMS Manifest contains IMS Simple Sequencing Information.
  • Verifies that the sequencing extensions are valid according to the SCORM 2004 4th Ed. Sequencing Extensions XML XSD Version 2.0 ( adlseq_v1p3.xsd found at http://www.adlnet.gov/ ) if the IMS Manifest contains SCORM CAM Sequencing extensions.
  • Verifies that the navigation extensions are valid according to the SCORM 2004 4th Ed. Navigation Extensions XML XSD Version 1.0 ( adlnav_v1p3.xsd found at http://www.adlnet.gov/ ) if the IMS Manifest contains SCORM CAM Navigation extensions.
Operational Instructions
  1. Provide the Manifest Identifier Name, Manifest Version/Release Number and Manifest Vendor/Developer information.
  2. Click Continue.
  3. Select the type of test to be performed by selecting the appropriate radio button. You can test an IMS Manifest in the form of a Package Interchange File (PIF), in the form of a PKZIP 2.0.4 (or compatible), or an uncompressed Package (non-PIF).
  4. Select the type of SCORM Content Package Application Profile to be tested (i.e., Content Aggregation Manifest or Resource Manifest) by selecting the appropriate radio button.
  5. Click Continue.
  6. Click Browse to select the imsmanifest.xml that is to be tested.
  7. After you click Begin Test the Test Suite will test the IMS Manifest and any inline metadata for schema validation only.

Additional Functionality
  • Log Legend: Clicking this link will open a window describing the meaning of the colors and symbols used in the Test Suite log.

Operational Notes
Both Summary and Detailed Logs are automatically saved in the folder:
<install_directory>\SCORM_2004_4th_Ed_TS_V1.1.1\TestSuite\Logs\CP\<timestamp-test subject name>\

Note: Logs are saved in this folder as a result of a portability enhancement. If you need to move the logs, move the entire folder and its contents since they are required to properly render the logs in the browser.
