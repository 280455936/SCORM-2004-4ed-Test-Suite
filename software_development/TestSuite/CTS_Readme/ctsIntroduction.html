Test Suite
Introduction
The SCORM 2004 4th Edition (Ed.) Test Suite Version 1.1.1 [Referred to hereafter as the Test Suite] allows organizations to perform self testing on learning management systems (LMSs), Sharable Content Objects (SCOs) and content packages. The Test Suite performs testing to help organizations determine their conformance to SCORM 2004 4th Ed. Note: This version of the Test Suite is built to assess conformance with SCORM 2004 4th Ed. The use of this software DOES NOT IMPLY certification to SCORM 2004 4th Ed. For details on conformance and/or certification, refer ADLNet.gov.

SCORM defines a web-based learning "Content Aggregation Model" and "Run-Time Environment" for content objects. SCORM integrates a set of related technical standards, specifications, and guidelines designed to meet SCORM's high-level requirements-accessible, interoperable, durable, and reusable content and systems. SCORM content can be delivered to your learners via any SCORM-conformant LMS using the same version of SCORM.

SCORM specifically addresses a common:
  1. Specification for a Content Aggregation Model which includes content packaging.
  2. Method for representing the intended behavior of an authored learning experience such that any LMS will sequence discrete learning activities in a consistent way.
  3. Specification for a Run-Time Environment (RTE) that facilitates interoperable communication between LMSs and course content components.
The Test Suite assesses SCORM conformance by providing four testing components:
  1. LMS Conformance Test
  2. Content Package Conformance Test
  3. SCO RTE Conformance Utility Test
  4. Manifest Utility Test.

The Test Suite provides a common framework for all of the conformance tests. Each test includes a step-by-step set of detailed instructions as well as a log of each test action and the outcome.

The Test Suite runs within a Web browser on one local computer. All of the software is loaded on the machine and requires no HTTP Web server to run.
