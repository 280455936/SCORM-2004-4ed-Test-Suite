Running the SCO RTE Conformance Utility Test

To run the SCO RTE Conformance Utility Test, click the corresponding link from the main Test Suite menu. A Java Plug-in Security Warning dialog box will appear because the Test Suite is reading and writing files from/to the local file system via a Java Applet. Click Grant this Session or Grant Always to allow the Test Suite applet to run. Clicking Deny will cause the Test Suite software to not function properly.

SCO Utility Overview
The SCO RTE Conformance Utility Test verifies that the test subject SCO is conformant with the SCORM 2004 4th Ed. Testing Requirements. This test will verify that the test subject:
  • Can be launched by a known conformant LMS

  • Supports the Run-Time Environment Application Program Interface (API) functions defined in SCORM 2004 4th Ed.

    Required
    • Initialize("")
    • Terminate("")
    Optional
    • Commit("")
    • GetValue("parameter")
    • SetValue("parameter_1","parameter_2")
    • GetLastError()
    • GetErrorString("parameter")
    • GetDiagnostics()
In addition to verification of the above requirements, this test will also audit which Run-Time Environment Data Model elements the SCO used in the content implementation, and verify that data elements are used in the correct manner with regard to the data type requirements associated with the data model elements.
Operational Instructions
  • The test can be run on a Single SCO or Multiple SCOs. You can also save or load a list of SCO file paths to and from a text (.txt) file.
    1. Provide the SCO Title, SCO Version/Release Number and SCO Vendor/Developer.
    2. Click Continue.
    3. Enter a timeout period (default is 20 seconds). This is the number of seconds the Test Suite will wait for the API method Initialize("") to be called. If the call is not made within this time, the Test Suite will end and the SCO will be given a conformance label.

      Note: If there is a possibility that the SCO may not call API method Initialize("") in the first 20 seconds, change this default value. No warning or error will be logged for changing this value.
    4. Click Browse to select the SCO that will be tested or a saved .txt file containing a list of SCO file paths.
    5. You may click Add to List to add another SCO or .txt file to the list.
    6. Click Continue after adding SCOs to the list.

      Note: If a file remains in the file input box, it will be appended to the list after you click Continue.

    7. Optionally, you may enter initialization values for any of the following elements: Learner ID, Learner Name, Credit/No-Credit, Mode, Comments From LMS, Launch Data, Objective Data, Scaled Passing Score, Max Time Allowed, Time Limit Action or Completion Threshold. These values are typically defaults that are provided by the LMS or via a Manifest.

      Note: If entering Comments From LMS or Objective Data, click the corresponding Add button, enter the values for the initial element of the array and click the corresponding Set button. After adding the last element of the list, click Done.

    8. Click Continue.
    9. Then click Launch SCO.
    10. Follow the necessary steps that are presented by the SCO to exercise the content.
    11. Click Complete Test in the Instructions frame to finish the test.
    12. If multiple SCOs were entered you will repeat steps 9 through 11 for each SCO to be tested.

  • After you click Launch SCO the Test Suite will launch the current SCO.

  • The Test Suite will then wait for a call to Initialize(""). The Test Suite will either wait for the call or the timeout period to expire (whichever comes first).

  • The Test Suite will wait for calls to any of the API functions listed below.

  • For Single SCO Testing: After the SCO RTE Conformance Utility Test surrogate LMS receives a call to Terminate(""), the SCO RTE Conformance Utility Test will end. Click Complete Test after this test ends to close the SCO window.

  • For Multiple SCO Testing: After the SCO RTE Conformance Utility Test surrogate LMS receives a call to Terminate(""), the current SCO RTE Conformance Utility Test will end. Click Complete Test to close the current SCO window. Processing will begin for the next SCO and will continue until the last SCO is tested.

  • Use Abort Test if a call to Terminate("") is not received from the SCO being tested. Wait for a period of time to see if the call will be invoked. If not, Abort Test will send a call to Terminate("") and end the testing, marking the SCO as being non-conformant.

  • At the end of a successful test the Test Suite will send to the Test Suite Log a set of supported features. These indicate what SCORM features a SCO supports. SCORM 2004 4th Ed. features are:

    API Support:
    • Find an LMS provided API Instance

    Session Methods:
    • Initialize()
    • Terminate()

    Data Transfer:
    • GetValue()
    • SetValue()
    • Commit()

    Support Methods:
    • GetLastError()
    • GetErrorString()
    • GetDiagnostic()

    Data Model Support:
    • SCORM 2004 4th Ed. Run-Time Environment Data Model
Additional Functionality
  • Log Legend: Clicking this link will open a window to describe the meaning of the colors and symbols used in the Test Suite log.

  • Saving a list of SCOs: To save a list of SCOs select all the SCOs you want in the list using the Browse and Add to List buttons. Select the .txt file you wish to save to or create a new .txt file using the Browse dialog. Click Save The List and you will be prompted to confirm the file to write. Click OK to save this file or Cancel to discard it.
    Note: If you want to edit the .txt file outside the SCORTE Instructions page open the file using a text editor. Each SCO path is entered on a separate line of the file. File paths are not validated so each line needs to be a valid path or the test will not work.

    Loading a list of SCOs: To load a list of SCOs that was created previously select that .txt file using the Browse button of step 2 to select it. Click Add To List or Continue to include the paths from that file to the list of SCOs (this action will append the list, not overwrite it). You may continue to add individual SCOs or load another saved list.
Operational Notes
Both Summary and Detailed Logs are automatically saved in the folder:
<install_directory>\SCORM_2004_4th_Ed_TS_V1.1.1\TestSuite\Logs\SCO\<timestamp-test subject name>\

Note: Logs are saved in this folder as a result of a portability enhancement. If you need to move logs, please move the entire folder and its contents since they are required to properly render the logs in the browser.

If using Internet Explorer, it is critical for Run-Time API execution that the SCOs be launched from the same URL domain as the SCO RTE Conformance Utility Test. The SCOs being tested will likely use some sort of scripting language (i.e. JavaScript) to locate the LMS's API Adapter object, which requires the use of cross frame and/or cross-window scripting. For security reasons Internet Explorer requires that content that performs cross frame or cross-window scripting be loaded from the same domain.
