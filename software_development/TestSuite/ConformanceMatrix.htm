ADL SCORM® Conformance Matrix

Learning Management System (LMS)
Conformance Label:
  • LMS SCORM 2004 4th Edition Conformant
The LMS shall adhere to the Testing Requirements defined for the following Conformance Categories:
  • LMS Run-Time Environment Version 1.1 (LMS RTE 1.1)

  • LMS Content Aggregation Model Version 1.1 (LMS CAM 1.1)

  • LMS Sequencing and Navigation Version 1.1 (LMS SN 1.1)
Conformance Category:
  • LMS RTE 1.1
Requirements Summary: The LMS shall:
  • Be able to launch an asset.
  • Be able to launch a known SCORM 2004 4th Edition Conformant SCO.
  • Provide and expose an API Instance as a Document Object Model (DOM) object that correctly implements all of the API methods.
  • Implement correct support for all SCORM 2004 4th Edition Run-Time Environment Data Model Elements.
  • Implement correct support for all SCORM 2004 4th Edition Navigation Data Model Elements.
Conformance Category:
  • LMS CAM 1.1
Requirements Summary: The LMS shall:
  • Be able to "import" and process a known Conformant SCORM 2004 4th Edition Content Aggregation Application Profile Content Package.
  • Initialize correctly SCORM 2004 4th Edition Run-Time Environment Data Model elements based on information supplied in a Content Package Manifest.
Conformance Category:
  • LMS SN 1.1
Requirements Summary: The LMS shall correctly implement:
  • All of the sequencing behaviors defined by the pseudo-code included in the SCORM 2004 4th Edition Sequencing and Navigation (SN) Version 1.1.
  • Support for all SCORM 2004 4th Edition Navigation Data Model Elements.
  • Support for Navigation User Interface requirements.


Sharable Content Object (SCO)
Conformance Label:
  • SCO SCORM 2004 4th Edition Conformant
The SCO shall adhere to the Testing Requirements defined for the following Conformance Categories:
  • SCO Run-Time Environment Version 1.1 (SCO RTE 1.1)

Conformance Category:
  • SCO RTE 1.1
Requirements Summary
  • The SCO shall search for and find an API Instance named API_1484_11 as a Document Object Model (DOM) object as defined in the SCORM 2004 4th Edition Run-Time Environment Version 1.1.
  • The SCO shall successfully invoke, at a minimum, the Initialize("") and Terminate("") API methods.
  • If used, the SCO shall successfully invoke the Data Transfer Methods.
  • If used, the SCO shall successfully invoke the Support Methods.
  • If using the Data Transfer Methods, the SCO shall ensure that all SCORM 2004 4th Edition Run-Time Environment Data Model elements used in the method calls adhere to the requirements of those elements.


Content Package
Conformance Label:
  • CP SCORM 2004 4th Edition Conformant
The content package shall adhere to the Testing Requirements defined for the following Conformance Categories:
  • Content Package Content Aggregation Model Version 1.1 (CP CAM 1.1)

  • Content Package Run-Time Environment Version 1.1 (CP RTE 1.1)
Conformance Category:
  • CP CAM 1.1
Requirements Summary
  • The content package shall conform to the requirements defined for the content package.
  • If the content package is a SCORM 2004 4th Edition Content Aggregation Content Package Application Profile, then the manifest shall conform to the Content Aggregation Content Package Application Profile Manifest requirements.
  • If the content package is a SCORM 2004 4th Edition Content Aggregation Content Package Application Profile and the manifest contains SCORM 2004 4th Edition Sequencing information, then the sequencing extensions in the manifest shall conform to the SCORM 2004 4th Edition Sequencing Extension requirements.
  • If the content package is a SCORM 2004 4th Edition Content Aggregation content package Application Profile and the manifest contains SCORM 2004 4th Edition Navigation/Presentation information, then the navigation/presentation extensions in the manifest shall conform to the SCORM 2004 4th Edition Navigation/Presentation Extension requirements.
  • If the content package is a SCORM 2004 4th Edition Resource Content Package Application Profile, then the manifest shall conform to the Resource Content Package Application Profile Manifest requirements.
  • If the content package manifest contains metadata, then the metadata instance shall be well-formed and valid according to the corresponding provided Controlling Document (e.g., XSD, DTD).
Conformance Category:
  • CP RTE 1.1
Requirements Summary
  • The content package shall contain at least one SCO resource or asset resource.
  • All SCO resources identified in the manifest shall conform with the SCO Testing Requirements.
